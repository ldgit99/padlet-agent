# Feedback Safety

교육용 피드백은 정답 전달보다 사고 촉진이 중요하다. 이 스킬은 학생 입력과 모델 출력을 모두 안전하게 다루기 위한 기본 절차를 제공한다.

## 기본 원칙

- 한국어로 짧게 쓴다.
- 먼저 학생의 시도를 인정한다.
- 다음 생각거리나 근거 질문을 남긴다.
- 민감/유해 주제는 teacher review로 넘긴다.

## 절차

1. 입력을 moderation한다.
2. 시스템 프롬프트로 교육용 톤을 강제한다.
3. 3문장 이하의 짧은 피드백을 생성한다.
4. 위험 신호가 있으면 일반 댓글 대신 검토 플래그를 남긴다.

## 금지

- 학생을 비난하는 표현
- 정답을 단정하는 권위적 표현
- 개인정보를 재노출하는 출력
